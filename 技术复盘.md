### 1.2 分层设计

项目分为三个清晰的层：

- **界面层**：CLI（`main.cpp`）和 GUI（`gui.cpp`），负责与用户交互，互不依赖
- **计算层**（`integrator.h/cpp`）：纯数值积分逻辑，不关心表达式来源，接收 `std::function<double(double)>` 和区间参数，返回 `IntegrationResult`
- **解析层**（`parser.h`）：封装 exprtk，将字符串表达式转换为可调用对象

这种分层的好处：
- 计算层**完全独立**于界面层，可以在 CLI 和 GUI 之间无缝共享
- 更换表达式解析库只需修改 `parser.h`，不影响其他模块
- 积分算法可以独立测试，不需要构造整个应用

### 1.3 接口设计

```cpp
// 积分器接口 — 接收任意可调用对象
IntegrationResult integrate(
    const std::function<double(double)>& f,  // 被积函数
    double a, double b,                       // 积分区间
    IntegrationMethod method,                 // 方法枚举
    int n,                                    // 分段数（非自适应方法）
    double tol                                // 误差容限（自适应方法）
);

// 返回值封装
struct IntegrationResult {
    double result;       // 积分结果
    int evaluations;     // 函数求值次数
    std::string method_name;  // 方法名称
};
```

`std::function<double(double)>` 作为接口核心，使调用方可以传入 lambda、函数指针、或任何可调用对象，而不需要关心具体的解析实现。

## 2. 数学原理角度

### 2.1 问题定义

定积分的数值计算问题：给定函数 $f(x)$ 和区间 $[a, b]$，求

$$I = \int_a^b f(x) \, dx$$

当 $f(x)$ 的原函数 $F(x)$ 无法用初等函数表示（如 $e^{-x^2}$、$\frac{\sin x}{x}$ 等），或原函数过于复杂时，数值方法成为必要选择。

### 2.2 梯形法 (Trapezoidal Rule)

**原理**：将区间 $[a, b]$ 等分为 $n$ 个子区间，在每个子区间上用线性插值（梯形）近似积分。

$$\int_a^b f(x) dx \approx \frac{h}{2} \left[ f(a) + 2\sum_{i=1}^{n-1} f(a+ih) + f(b) \right]$$

其中 $h = \frac{b-a}{n}$。

**误差分析**：
- 截断误差：$E_T = -\frac{(b-a)h^2}{12} f''(\xi)$，$\xi \in [a, b]$
- 精度：$O(1/n^2)$ — 分段数翻倍，误差约减为 $1/4$
- 对 $f''(x)$ 有要求：函数越"弯曲"，误差越大

**优缺点**：
- 实现最简单，计算量小
- 收敛速度慢，需要很多分段才能达到高精度

### 2.3 Simpson 1/3 法则 (Simpson's Rule)

**原理**：在每个相邻子区间对上使用**二次插值**（抛物线）近似积分。需要 $n$ 为偶数。

$$\int_a^b f(x) dx \approx \frac{h}{3} \left[ f(a) + 4\sum_{i=1}^{n/2} f(a+(2i-1)h) + 2\sum_{i=1}^{n/2-1} f(a+2ih) + f(b) \right]$$

**几何意义**：每两个相邻小区间构成一个抛物线，计算抛物线下的面积。

**误差分析**：
- 截断误差：$E_S = -\frac{(b-a)h^4}{180} f^{(4)}(\xi)$，$\xi \in [a, b]$
- 精度：$O(1/n^4)$ — 分段数翻倍，误差约减为 $1/16$
- 对三次多项式可**精确积分**（因为 $f^{(4)}(x)=0$）

### 2.4 自适应 Simpson 法 (Adaptive Simpson)

**核心思想**：不用均匀网格，而是在函数变化剧烈的区域自动加密，在平滑区域稀疏采样。

**递归细分策略**：

1. 在 $[a, b]$ 上用 Simpson 公式计算 $S(a, b)$
2. 在中点 $m = (a+b)/2$ 处分割，分别计算 $S(a, m)$ 和 $S(m, b)$
3. 误差估计：$|S(a, m) + S(m, b) - S(a, b)|$
4. 若误差 $< 15 \cdot \text{tol}$，则认为达到精度，返回 $S(a, m) + S(m, b) + \frac{\text{误差}}{15}$（Richardson 外推修正）
5. 否则对两个子区间**递归**调用，容差减半

**为什么误差阈值是 $15 \cdot \text{tol}$？**

这来自 Richardson 外推。Simpson 法则的误差主要项为 $Ch^4$。对半分之后，$S(a,m) + S(m,b)$ 的误差约为 $2 \cdot C(h/2)^4 = Ch^4/8$，而 $S(a,b)$ 的误差为 $Ch^4$。两者之差约为 $(7/8)Ch^4$。真实误差约为 $|S(a,m) + S(m,b) - S(a,b)|/15$，当这个值小于 tol 时认为收敛。

**优势**：
- 自动适应函数复杂度，在尖峰区域加密
- 对平滑函数（如多项式）只需很少的求值次数

**代价**：
- 递归调用有栈开销
- 对高度振荡函数可能递归过深

### 2.5 三种方法对比

| 方法 | 收敛阶 | 实现复杂度 | 典型精度(n=1000) | 适用场景 |
|------|--------|-----------|-------------------|---------|
| 梯形法 | O(1/n²) | 低 | ~6位 | 函数平滑，快速估算 |
| Simpson | O(1/n⁴) | 中 | ~10位 | 函数光滑，精度要求高 |
| 自适应 | 变阶 | 高 | 自动达 tol | 函数变化剧烈，需要误差保证 |

## 3. 工具链角度

### 3.1 编译器：MinGW-w64 g++ 15.2.0

**定位**：Windows 平台上的 GCC 移植版，提供 POSIX 兼容层和 Win32 原生 API 支持。

**关键选项**：

| 选项 | 作用 |
|------|------|
| `-std=c++17` | 启用 C++17 标准（exprtk 需 C++17 以上） |
| `-O2` | 二级优化，平衡编译时间和运行性能 |
| `-Wa,-mbig-obj` | 允许生成超过 65535 个节（section）的 COFF 目标文件 |
| `-mwindows` | GUI 子系统，不显示控制台窗口 |

**为什么需要 `-Wa,-mbig-obj`？**

exprtk.hpp 是一个约 1.6 MB 的单头文件库，内部大量使用模板元编程技术。展开后的目标文件包含数万个 COFF section（每个模板实例化对应一个 section），超出默认的 65535 上限。`-mbig-obj` 是 MinGW 提供的扩展，允许更大数量的 section。

### 3.2 外部库：exprtk

**全称**：C++ Mathematical Expression Toolkit Library

**作者**：Arash Partow（1999-2025）

**设计特点**：
- 纯头文件（header-only），无需编译
- 使用编译器内部的**递归下降解析器**生成表达式树
- 支持向量化、编译时表达式优化
- 支持自定义符号表和常量

**内部工作流程**：

```
字符串表达式 (如 "sin(x) + x^2")
    │
    ▼ 词法分析 (Lexer)
Token 序列 (sin, (, x, ), +, x, ^, 2)
    │
    ▼ 语法分析 (Parser)
抽象语法树 (AST)
    │       ┌─── + ───┐
    │       │         │
    │     sin        ^
    │       │       / \
    │       x      x   2
    │
    ▼ 求值 (Evaluator)
后序遍历 AST，逐节点计算 → double
```

## 4. 算法实现角度

### 4.1 梯形法实现

```cpp
double trapezoidal(const std::function<double(double)>& f, double a, double b, int n, int& evals) {
    double h = (b - a) / n;
    double sum = 0.5 * (f(a) + f(b));
    evals += 2;

    for (int i = 1; i < n; ++i) {
        sum += f(a + i * h);
        evals += 1;
    }

    return sum * h;
}
```

**实现要点**：
- 先计算端点值（带系数 1/2），再加内点，避免分支
- 使用 `int& evals` 引用计数函数求值次数
- 直接在循环中计算 `a + i*h`（而非递推累加），避免浮点误差累积

### 4.2 Simpson 法实现

```cpp
double simpson(const std::function<double(double)>& f, double a, double b, int n, int& evals) {
    if (n % 2 != 0) n++;  // 确保偶数

    double h = (b - a) / n;
    double sum = f(a) + f(b);
    evals += 2;

    for (int i = 1; i < n; ++i) {
        double x = a + i * h;
        sum += (i % 2 == 0) ? 2.0 * f(x) : 4.0 * f(x);
        evals += 1;
    }

    return sum * h / 3.0;
}
```

**实现要点**：
- 自动修正奇数 `n` 为偶数（`n++`），对用户透明
- 奇偶位系数分离：奇数索引乘 4，偶数索引乘 2（端点已单独处理）
- 最后乘 `h/3`，而非在循环中操作

### 4.3 自适应 Simpson 实现

```cpp
static double adaptive_simpson_recursive(
    const std::function<double(double)>& f,
    double a, double b,
    double fa, double fb, double fm,  // 缓存端点值，避免重复求值
    double S,                          // 当前区间的 Simpson 近似
    double tol,
    int& evals)
{
    double m = (a + b) / 2.0;
    double lm = (a + m) / 2.0;
    double rm = (m + b) / 2.0;

    double fl = f(lm);
    double fr = f(rm);
    evals += 2;

    double S_left  = (m - a) / 6.0 * (fa + 4.0 * fl + fm);
    double S_right = (b - m) / 6.0 * (fm + 4.0 * fr + fb);

    if (std::abs(S_left + S_right - S) < 15.0 * tol) {
        return S_left + S_right + (S_left + S_right - S) / 15.0;
    }

    return adaptive_simpson_recursive(f, a, m, fa, fm, fl, S_left,  tol / 2.0, evals)
         + adaptive_simpson_recursive(f, m, b, fm, fb, fr, S_right, tol / 2.0, evals);
}
```

**实现要点**：

1. **缓存求值结果**：`fa`, `fb`, `fm` 作为参数传递，避免在递归中重复计算 $f(a)$、$f(b)$、$f(m)$，这是经典的性能优化

2. **误差修正**：`return S_left + S_right + (S_left + S_right - S) / 15.0`
   利用 Richardson 外推提高一阶精度。$(S_left + S_right - S)/15$ 是对剩余误差的估计，加到结果上相当于对 Simpson 值做了一次校正

3. **容差减半**：`tol / 2.0` 保证子区间的误差之和不超过原始 tol。这是一个保守策略，确保全局误差可控

4. **递归终止**：当区间很小或函数足够平滑时触发终止。无显式的最大深度限制——对极端振荡函数可能栈溢出，但在实用场景中很少出现

### 4.4 区间异常处理

```cpp
// 零区间
if (a == b) {
    result.result = 0.0;
    return result;
}

// 交换区间
if (a > b) {
    std::swap(a, b);
    auto r = integrate(f, a, b, method, n, tol);
    result.result = -r.result;  // ∫_b^a f = -∫_a^b f
    return result;
}
```

利用定积分的**方向性**性质：$\int_a^b f = -\int_b^a f$，自动处理上下限互换。

### 4.5 表达式解析封装

```cpp
class MathParser {
public:
    MathParser() {
        symbol_table_.add_variable("x", x_);
        symbol_table_.add_constants();   // pi, e, epsilon 等
        expression_.register_symbol_table(symbol_table_);
    }

    bool compile(std::string_view expr) {
        return parser_.compile(std::string(expr), expression_);
    }

    double evaluate(double x_val) {
        x_ = x_val;             // 更新 x 变量值
        return expression_.value();  // 重新求值
    }
};
```

**设计思路**：
- 编译与求值分离：`compile` 只做一次语法分析，`evaluate` 可多次调用
- `x_` 通过指针绑定到符号表，修改 `x_` 即更新表达式中 `x` 的值，无需重新编译
- 这是 exprtk 的高效用法——避免了每次求值都重新解析字符串

## 5. 遇到的问题与解决方案

### 5.1 大目标文件 section 溢出

**现象**：
```
build/main.o: too many sections (76180)
build/main.o: file too big
```

**原因**：exprtk.hpp 的模板展开产生数万个 COFF section，超过 MinGW 默认限制。

**解决**：添加编译标志 `-Wa,-mbig-obj`，启用 COFF 大目标格式支持。

### 5.2 中文乱码

**现象**：GUI 窗口中的中文标签显示为乱码。

**原因**：源代码使用 UTF-8 编码，但 `CreateWindowExA` 等 ANSI API 按照系统活动代码页（GBK/CP936）解释字符串，UTF-8 字节序列被错误解读。

**解决**：
- 定义 `UNICODE` / `_UNICODE`，使用 Unicode API
- 用 `L"被积函数"` 宽字符串字面量代替 `"被积函数"`
- 在 Win32（UTF-16）和计算引擎（UTF-8）之间用 `WideCharToMultiByte` / `MultiByteToWideChar` 转换

### 5.3 exprtk 下载缓慢

**现象**：curl 下载 exprtk.hpp（1.6 MB）超时。

**原因**：raw.githubusercontent.com 连接速度慢。

**解决**：改用 Python 的 `urllib.request.urlretrieve` 下载，成功获取完整文件。

### 5.4 make 命令不可用

**现象**：`make` 报 command not found。

**原因**：MinGW 的 make 命名为 `mingw32-make` 以区别于 MSYS 的 make。

**解决**：使用 `mingw32-make` 代替 `make`。

## 6. 可改进方向

### 6.1 算法扩展

- **Romberg 积分法**：基于 Richardson 外推的迭代方法，收敛极快
- **Gauss-Legendre 求积**：使用最优节点的高斯求积法，相同求值次数下精度最高
- **多重积分**：扩展到二维、三维积分（需要新的适配层）
- **分段自适应策略**：对不同的光滑度采用不同的阈值，或结合全局误差控制

### 6.2 工程改进

- **单元测试**：引入 Catch2 或 Google Test，覆盖边界情况和已知积分结果
- **性能优化**：使用 `-flto`（链接时优化）减小二进制体积；对自适应递归做尾调用优化
- **表达式缓存**：支持多变量表达式（x, y, z）
- **输出格式**：支持输出中间过程（各子区间结果）

### 6.3 GUI 增强

- **函数图像绘制**：在结果区域显示被积函数的曲线和高亮积分区域
- **实时预览**：输入表达式时实时计算简单点的函数值，辅助验证
- **历史记录**：保存计算历史，方便对比不同方法的结果
- **导出功能**：将结果导出为 CSV 或 LaTeX 格式

### 6.4 部署相关

- **静态链接**：链接 `-static-libgcc -static-libstdc++` 消除 MinGW DLL 依赖
- **减小体积**：使用 `-Os` 优化体积，strip 调试符号
- **安装包**：使用 NSIS/Inno Setup 制作安装程序
